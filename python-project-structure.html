Python project structure

Published: Sun 08 September 2019
By brl0

In python.

tags: python

Python project structure is maddening and confusing.

Here is what I am trying to use:

  • my_project\
    • my_project\
      • my_project.py
      • __init__.py
      • __main__.py
      • app.py
      • cli.py
    • LICENSE
    • README.*
    • requirements.txt
    • dev-requirements.txt
    • setup.py
    • docs\
    • tests\
      • test_my_project.py
  • my_project - Root project folder, git should sync here.
  • my_project (2) - Python package root, contains modules and submodules, etc.
  • my_project.py - Intuitive entry point for script execution.
  • __init__.py - File necessary for python to recognize package.
  • __main__.py - Entry point for modules executed via python -m my_project.
